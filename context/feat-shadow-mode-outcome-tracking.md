feat(outcomes): add shadow-mode signal outcome tracking (CB-42)

## Summary

Adds optional, fail-open shadow-mode outcome tracking for trading signals generated by the bot. When enabled via `ENABLE_SIGNAL_OUTCOME_TRACKING=true`, signals are recorded to Firestore with normalized metadata, and aggregate metrics (hit rate, average return, MAE/MFE) can be queried via a new protected endpoint.

## Key Changes

### 🗄️ New `SignalOutcomeService` (`src/services/storage/SignalOutcomeService.js`)

- Feature-gated via `ENABLE_SIGNAL_OUTCOME_TRACKING=true` (disabled by default)
- Fail-open: Firestore errors are caught/logged — alert delivery is **never blocked**
- Reuses the existing Firebase Admin singleton (no new SDK dependencies)
- Stores signals in the `signal-outcomes` Firestore collection with:
  - Normalized metadata: `symbol`, `exchange`, `timeframe`, `setupType`, `side`, `score`, `entryPrice`, `stopPrice`, `targetPrice`
  - Future outcome windows: `1h`, `4h`, `1D`, `1W` (initialized as `null`, to be filled by an evaluator)
  - `shadowMode: true` tag on every record so it's clearly diagnostic-only
- `computeMetrics()` function computes hit rate, average return, MAE, MFE per time window
- Missing market data in outcome windows is treated as **unavailable** (not zero)

### 🌐 New Outcomes Controller (`src/controllers/outcomes/outcomes.js`)

- `GET /api/outcomes/summary` — returns aggregate metrics over recorded signals
- Protected by `validateApiKey`
- Returns `403 FEATURE_DISABLED` when feature is off
- Returns `503 STORAGE_UNAVAILABLE` when Firestore is enabled but unreadable
- Supports query params: `limit` (max 500), `source`, `symbol`

### 🔀 Route Registration (`src/routes/index.js`)

- Registers `GET /api/outcomes/summary` behind `validateApiKey`

### 📊 Status Visibility (`src/controllers/status.js`)

- Adds `signalOutcomeTracking` boolean to `featureFlags` in `GET /api/status` response

## Technical Implementation

### Architecture

The service follows the same lazy Firestore singleton pattern as `AlertStorageService` and `NewsDedupStorageService` — no new Firestore initialization code is needed. The service skips all I/O when the feature flag is off.

### File Structure Additions

```
src/services/storage/SignalOutcomeService.js   — core service

```
